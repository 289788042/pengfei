# 错位时卡 - 开发文档

> **项目名称**：错位时卡  
> **引擎版本**：Godot 4.6.2 (Forward+ 渲染器)  
> **分辨率**：1920×1080 (窗口 1280×720)  
> **主场景**：`res://scenes/StartMenu.tscn`  
> **编程语言**：GDScript 4.x  

---

## 一、项目概述

### 1.1 游戏类型

**都市女性生存模拟器** —— 以智能手机 UI 为核心载体的文字策略游戏。玩家扮演一个 23 岁南下深圳打拼的女孩，在"深漂"生活中管理金钱、精力、情绪、颜值、学识、情商六大属性，通过微信社交、消费决策、职业选择等多线交互，体验当代都市女性的生存困境。

### 1.2 核心设计理念

- **手机即世界**：整个游戏界面就是一部智能手机，所有操作都通过手机上的 App 完成
- **属性即人生**：六大核心属性直接决定可用的选项和事件走向
- **消费即叙事**：花呗分期、冲动消费、理财亏损等真实经济行为驱动剧情
- **NPC 即关系**：微信联系人系统模拟真实的社交关系网络，好感度+Flag 驱动多结局

### 1.3 游戏流程概览

```
开始菜单 (StartMenu.tscn)
  → 输入姓名
  → 选择星象（土/火/水/风 → 初始属性加成）
  → 滑动解锁手机动画
  → 进入主游戏 (MainGame.tscn)
	  → 周循环：工作日(吃饭→上班) → 周末(自由活动)
	  → 月循环：4周结束 → 月度账单结算
	  → 年循环：12月结束 → 春节回家 → 长一岁
	  → 35岁 → 结局判定
```

### 2.2 模块依赖关系

```
GameManager (Autoload 单例)  ←── 所有模块读写核心数值
	↑
NPCManager (Autoload 单例)   ←── WeChatSystem / GalgameSystem 查询 NPC 配置
	↑
MainGame (场景控制器)
	├── GalgameSystem  (RefCounted，通过 load().new() 创建)
	├── AlipaySystem   (RefCounted)
	├── WeChatSystem   (RefCounted)
	├── AppPopupSystem (RefCounted)
	└── DebugPanel     (作为子节点动态创建)
```

### 2.3 架构设计要点

**为什么子系统用 `RefCounted` 而不是 `Node`？**
- 子系统（GalgameSystem、AlipaySystem 等）不需要加入场景树
- 通过 `load("res://scripts/XXX.gd").new()` 动态创建，避免 `class_name` 的 Godot 缓存冲突
- 子系统持有 `_main: Node` 引用来访问 MainGame 的 UI 节点

**为什么不用 `class_name`？**
- Godot 4.6 对 `class_name` 的热重载支持有限
- 多个脚本使用 `class_name` 后，修改任意一个会导致其他脚本的缓存失效
- 使用 `load()` + 手动 `.new()` 的模式完全规避了这个问题

## 三、全局单例详解

### 3.1 GameManager（`res://Scripts/GameManager.gd`）

**职责**：核心数值存取、时间推进、月度结算、结局判定、NPC 数据管理

#### 3.1.1 Autoload 注册

在 `project.godot` 中以 `*` 前缀注册（全局可用）：
```
GameManager = *res://Scripts/GameManager.gd
```

#### 3.1.2 信号系统

| 信号 | 参数 | 触发时机 |
|------|------|---------|
| `stats_updated` | 无 | 任何属性变化后 |
| `week_advanced(new_week)` | 总周数 | 每周推进后 |
| `game_over(cause_title, cause_desc)` | 死法标题+描述 | 触发死亡结局 |
| `game_ended(ending_type)` | 结局类型 | 35岁结局判定 |
| `npc_unlocked(npc_id, npc_name)` | NPC ID+名称 | 解锁新 NPC |
| `month_ended(pending_salary, rent_cost, huabei_debt, food_cost)` | 月度财务数据 | 月底到来 |
| `monthly_settled(net_change)` | 净收支 | 月度结算完成 |
| `invest_settled(safe_profit, risk_profit)` | 理财收益 | 月度理财结算 |
| `aging_decayed` | 无 | 每月颜值衰减 |
| `spring_festival(msg)` | 春节消息 | 每年春节 |

#### 3.1.3 核心属性

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `age` | int | 23 | 当前年龄 |
| `month` | int | 1 | 当前月份 (1-12) |
| `week_in_month` | int | 1 | 月内周数 (1-4) |
| `turn_count` | int | 1 | 总周数 |
| `money` | int | 3000 | 初始金钱（制造"虚假繁荣"） |
| `pending_salary` | int | 0 | 待发工资 |
| `energy` | int | 100 | 精力 |
| `max_energy` | int | 100 | 精力上限（火象→120） |
| `sanity` | int | 100 | 情绪 |
| `max_sanity` | int | 100 | 情绪上限（水象→120） |
| `charm` | int | 10 | 颜值 |
| `intellect` | int | 10 | 学识 |
| `eq` | int | 10 | 情商 |
| `huabei_debt` | int | 0 | 花呗欠款（未分期） |
| `huabei_installment_debt` | int | 0 | 花呗分期本金 |
| `huabei_installment_months_left` | int | 0 | 分期剩余月数 |
| `invest_safe` | int | 0 | 稳健理财本金 |
| `invest_risk` | int | 0 | 高风险基金本金 |
| `housing_level` | int | 0 | 住房等级 (0城中村/1精装/2大平层) |
| `degree` | int | 0 | 学历等级 (0大专/1本科) |
| `job_level` | int | 0 | 职位等级 (0行政/1运营/2经理) |
| `night_school_progress` | int | 0 | 夜校学分 (满12=本科) |
| `guilt_debuff_weeks` | int | 0 | 愧疚 debuff 剩余周数 |
| `mom_care_buff_weeks` | int | 0 | 妈妈关爱 buff 剩余周数 |
| `consecutive_overtime` | int | 0 | 连续加班周数（满5=猝死） |
| `consecutive_poor_food` | int | 0 | 连续吃土周数（满5=营养不良） |

#### 3.1.4 核心方法

**`modify_stat(stat_name: String, amount: int)`**
- 修改指定属性，自动做边界检查
- `energy` 限制在 `[0, max_energy]`
- `sanity` 限制在 `[0, max_sanity]`，sanity≤0 触发死亡
- `money` 限制在 `[0, ∞)`（不会变负数，破产由月度结算处理）
- 修改后自动发射 `stats_updated`

**`advance_week()`**
- 每周推进的核心入口
- 精力恢复至上限
- 房屋情绪恢复加成
- 愧疚 debuff 扣情绪
- 妈妈关爱 buff 加精力
- 月底（第5周）触发 `month_ended` 信号暂停推进

**`start_new_month()`**
- 月度结算确认后调用
- 严格按5步执行：
  1. 理财结算（稳健+5%/月，高风险-30%~+40%/月）
  2. 发放工资
  3. 扣除房租+餐饮
  4. 处理花呗分期/最低还款（未还部分月息5%）
  5. 破产惩罚（sanity-50）
- 颜值-1（岁月催人老）
- 12月结束 → 春节回家（固定扣5000）→ 长一岁
- 35岁 → 结局判定

**`check_behavior_death() -> Dictionary`**
- 检查连续行为死法
- 连续加班≥5周 → "福报的代价"（猝死）
- 连续吃土≥5周 → "恩格尔系数 0.01"（营养不良）

**`get_projected_balance() -> int`**
- 计算预计月末资产（含理财 - 房租 - 花呗 - 餐饮）
- 用于支付宝账本显示

#### 3.1.5 NPC 数据管理

| 方法 | 说明 |
|------|------|
| `load_npc_data()` | 从 `npc_data.json` 加载静态剧本数据 |
| `get_npc_data(npc_id)` | 获取指定 NPC 的静态剧本 |
| `get_npc_runtime(npc_id)` | 获取/创建运行时动态数据（好感度、flags、已用对话） |
| `is_npc_unlocked(npc_id)` | 检查 NPC 是否已解锁 |
| `unlock_npc(npc_id)` | 解锁 NPC（同步到微信联系人） |
| `add_unread(npc_id, count)` | 增加未读消息数 |
| `clear_unread(npc_id)` | 清除未读消息 |

**职责**：NPC 配置注册表，提供头像颜色、自动回复、朋友圈数据等查询接口

get_auto_reply(npc_id) -> String      # 获取随机自动回复
get_moments_text(npc_id) -> String    # 获取朋友圈文本

#### 4.2.2 时间循环（Phase 枚举）

| Phase | 说明 |
|-------|------|
| `WEEKDAY` | 工作日：先选饮食 → 再选工作模式 |
| `WEEKEND` | 周末：自由活动（地图/微信/App），点"下一周"推进 |
| `EVENT` | 随机事件弹窗（30%概率） |
| `MONTH_END` | 月度账单结算弹窗 |
| `TRANSITION` | 周间过渡黑屏 |
| `ENDING` | 结局画面 |
| `GAME_OVER` | 死亡画面 |

#### 4.2.3 周循环详细流程

```
工作日 (WEEKDAY)
  ├─ 第1步：选择饮食
  │   ├─ 挂逼套餐：+300餐饮, -15情绪, 连续吃土+1
  │   ├─ 打工人标配：+800餐饮, +10精力
  │   └─ 小资高档：+2000餐饮, +20情绪, +15精力
  ├─ 第2步：选择工作模式
  │   ├─ 正常上班：随机工资, 正常精力消耗
  │   ├─ 摸鱼：低工资, 少精力消耗
  │   └─ 疯狂加班：高工资, 大精力消耗, 连续加班+1
  └─ 自动进入周末

周末 (WEEKEND)
  ├─ 自由活动（App 全开放）
  │   ├─ 高德地图 → 图书馆/健身房/酒吧/宅家
  │   ├─ 微信 → 聊天/约会/家庭群
  │   ├─ 支服了宝 → 理财/花呗还款
  │   ├─ 宝淘/团美 → 消费提升颜值
  │   └─ 其他 App...
  ├─ 30% 概率触发随机事件
  ├─ 20% 概率触发深夜失眠
  └─ 点击"下一周" → advance_week()
```

#### 4.2.4 工资计算

```gdscript
# job_level 对应的工资范围
0: 初级行政 → randi_range(800, 2500)
1: 新媒体运营 → randi_range(2000, 6000)  # 需学识≥30
2: 大客户经理 → randi_range(4000, 12000) # 需本科学历+30岁以下

# 工作模式修正
摸鱼: salary *= 0.6
疯狂加班: salary *= 1.8, 精力额外-30, 连续加班+1
```

#### 4.2.5 进度条系统

属性面板使用 `GridContainer`（2列布局），每个属性行由 `HBoxContainer` 包裹 `ProgressBar + Label`：

```
[标题标签] [HBoxContainer: ProgressBar + 数值标签]
```

- 精力/情绪：显示 `当前/上限` 格式（如 `120/120`）
- 颜值/学识/情商：只显示当前值（如 `15`）
- 金钱：纯数字，无进度条
- 颜色：精力绿、情绪紫、颜值粉、学识橙、情商黄

实现方式：`_ready()` 中动态替换原有数值 Label 为 HBoxContainer（含进度条+数值标签），`_refresh_ui()` 中同步更新。

## 五、子系统详解

### 5.1 GalgameSystem（`res://scripts/GalgameSystem.gd`）

**职责**：分页对话、飘字动画、邂逅系统

#### 核心功能

| 功能 | 方法 | 说明 |
|------|------|------|
| 飘字 | `show_floating_text()` | 屏幕上方飘出带颜色的文字（绿=正面/红=负面），1秒淡出 |
| 消息提示 | `show_message()` | 底部对话框显示文本，3秒后淡出 |
| 紧急提示 | `show_urgent_message()` | 红色醒目提示，4秒淡出 |
| 分页对话 | `show_galgame_dialog(pages)` | 逐页打字机输出，点击翻页，支持回调 |
| 点击处理 | `gal_on_click()` | 跳过打字 → 显示全文；再点击 → 翻页 |
| 邂逅系统 | `start_wechat_request_phase()` | NPC 请求加微信 → 玩家选择 → 属性消耗 |

#### 打字机效果

- 每字间隔 0.03 秒
- 旁白文本：白色
- 对话文本（以"陌生男子："或"我："开头）：黄色
- 打字完成时显示 ▼ 箭头浮动指示器

#### 邂逅三阶段

1. **Phase 1 - 场景+对话+考验**：Galgame 分页播放
2. **Phase 2 - 请求加微信**：NPC 台词 + 玩家选项按钮
3. **Phase 3 - 选择结果**：属性消耗/变化 + flag 记录

### 5.2 AlipaySystem（`res://scripts/AlipaySystem.gd`）

**职责**：支付拦截、支付宝 UI、理财、花呗

#### 通用支付拦截

```gdscript
alipay.request_payment(cost, desc, category, on_success)
```

- 显示支付弹窗：现金支付 / 花呗透支 / 取消
- 现金支付：直接扣 money
- 花呗透支：增加 huabei_debt，不扣现金
- 取消：回调不执行

#### 理财产品

| 产品 | 月收益率 | 说明 |
|------|---------|------|
| 稳健宝 | +5% | 稳定收益，每次存入/取出 500 |
| 比特币/高风险 | -30%~+40% | 随机收益，可能亏损 |

#### 花呗还款

- **主动还款**：输入金额 → 先还未分期 → 再还分期本金（分期本金还清则自动结束分期）
- **最低还款**：月度结算时自动执行，`min(debt×10%+200, debt)`，未还部分月息5%
- **账单分期**：欠款≥3000时可办理，12期+15%总手续费

#### 支付宝 UI 显示

- 活期余额
- 花呗总欠款（未分期+分期本金）
- 稳健宝/高风险基金余额
- 预计月末资产（含理财-房租-花呗-餐饮）
- 分期信息（剩余期数/每月扣款/手续费）
- 财务流水日志（最近50条）

### 5.3 WeChatSystem（`res://scripts/WeChatSystem.gd`）

**职责**：微信聊天、联系人、朋友圈、家庭群

#### 微信 UI 结构

```
微信面板 (WeChatMenu)
├── 标题栏："微信 (N)" + 关闭按钮
├── Tab 栏：聊天 | 联系人 | 朋友圈
├── 聊天列表视图 (WCChatListView)
│   └── 动态生成的聊天条目（头像+名称+预览+未读红标）
├── 联系人视图 (WCContactsView)
│   └── 头像+名称+等级+好感度
├── 朋友圈视图 (WCMomentsContent)
│   └── NPC 动态卡片（头像+文字+点赞数）
└── 聊天详情视图 (WCChatView)
	├── 标题栏：NPC名称 + 好感度/亲情
	├── 消息区域（聊天气泡，左=NPC/右=自己）
	└── 输入按钮 → 打开操作菜单
```

#### 聊天菜单选项

| NPC | 可用操作 |
|-----|---------|
| 通用 | 日常闲聊(-10精力)、删除好友 |
| 家庭群 | 查看家庭消息（随机事件/闲聊） |
| 王老师 | 报名冲刺班(-50精力, -1000金, +1学分)、约会(等级≥2) |
| 其他NPC | 约会(-50精力, -500金, 等级≥2) |

#### 日常闲聊系统（防重复抽卡）

1. 从 `npc_data.json` 的 `daily_chats` 中抽取未使用的对话
2. 显示 NPC 发来的文本（多行）
3. 生成回复选项按钮（可能带属性门槛，不满足则灰色禁用）
4. 玩家选择后：扣除资源 → 应用属性变化 → 记录 flag → 显示 NPC 回复
5. 已使用的对话 ID 记入 `runtime["used_daily_chats"]`，不会重复
6. 所有对话用完 → 显示"对方无回复"

#### 家庭群系统

**消息推送逻辑**：
- 第1周固定推送：相亲局事件
- 第5周固定推送：冰箱事件
- 其他周：70%概率随机推送

**事件类型**：
- **选择事件**：弹窗显示描述+选项，选择后应用效果（如相亲局、冰箱）
- **闲聊消息**：随机抽取家庭群旁白文本，自动应用效果
- **正面事件**：加情绪/抢红包

**事件效果可包含**：
- 属性变化（sanity, money）
- 亲情变化（affection_gain）
- Buff/Debuff 设置（妈妈关爱、愧疚感）

#### 约会系统

- 消耗：50精力 + 500金（通过支付拦截）
- 效果：+40情绪, +30好感
- 需要NPC等级≥2

## 十、扩展指南

### 10.1 新增 NPC

1. 在 `npc_data.json` 中添加 NPC 条目（含 id、name、encounter、daily_chats、milestones）
2. 在 `NPCManager.gd` 的 `NPC_CONFIG` 中添加头像颜色和自动回复配置
3. 在 `GameManager.gd` 的 `npcs` 字典中添加初始条目（如需要预解锁）
4. 如需特殊聊天/约会逻辑，在 `WeChatSystem.gd` 的 `match` 分支中添加

### 10.2 新增地点

1. 在 `AppPopupSystem.gd` 的 `_on_app_map()` 方法的 `map_locs` 数组中添加条目
2. 实现对应的 `_on_loc_xxx()` 方法

### 10.3 新增 App

1. 在 `MainGame.tscn` 中添加 App 图标按钮
2. 在 `MainGame.gd` 中添加 `@onready` 引用和按钮信号连接
3. 在 `AppPopupSystem.gd` 中实现 App 逻辑
4. 如需覆盖层 UI，使用 `_build_app_overlay()` 通用构建器

### 10.4 新增死法

1. 在 `GameManager.gd` 中添加触发条件检查
2. 调用 `game_over.emit(title, desc)` 发射死亡信号
3. `MainGame.gd` 的 `_on_game_over()` 会自动显示死亡画面

### 10.5 新增日常聊天

1. 在 `npc_data.json` 对应 NPC 的 `daily_chats` 数组中添加新条目
2. 设置唯一 `id`（用于防重复）
3. 编写 `text_lines`（NPC 发言）和 `reply_options`（玩家选择）
4. 可选设置 `req_stat`/`req_val` 属性门槛
